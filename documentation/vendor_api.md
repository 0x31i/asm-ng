# ASM-NG External API — Vendor Integration Guide

This document is intended for **trusted external vendors** — including AI SOC agents, SIEM platforms, security orchestration tools, and automation pipelines — who have been granted API access to an ASM-NG deployment.

The External API (`/v1/ext`) is a read-only, scoped REST interface that exposes attack surface data: security grades, known assets, and validated findings for authorized targets. It is separate from the internal ASM-NG REST API and is designed for stable, long-term machine-to-machine integration.

## Quick Start

**Step 1 — Obtain your API key.** Contact your ASM-NG administrator. They will provision a key scoped to the targets and data types you need. Your key will look like:

```
asmng_ext_xK9mPq3r5sT7uV1wX2yZ4aBcDeFgHiJ
```

**Step 2 — Verify connectivity with a health check.**

```bash
curl -s https://asm.example.com/v1/ext/health
```

Expected response:

```json
{"status": "ok"}
```

**Step 3 — Make your first authenticated request.**

```bash
curl -s \
  -H "Authorization: Bearer asmng_ext_xK9mPq3r5sT7uV1wX2yZ4aBcDeFgHiJ" \
  https://asm.example.com/v1/ext/targets
```

If your key is valid and has the `assets:read` scope, you will receive a list of targets your key is authorized to access.

## Base URL

All External API endpoints are prefixed with `/v1/ext`. Requests must be made over HTTPS.

```
https://<your-asm-ng-host>/v1/ext
```

Your ASM-NG administrator will provide the exact hostname for your deployment.

## Authentication

Every request to the External API (except the health check) requires an `Authorization` header containing your Bearer token.

```
Authorization: Bearer asmng_ext_xK9mPq3r5sT7uV1wX2yZ4aBcDeFgHiJ
```

### Key format

API keys are prefixed with `asmng_ext_` followed by approximately 30 URL-safe random characters, providing roughly 240 bits of entropy. Keys are generated by the ASM-NG administrator and are non-recoverable — store them securely.

### Key properties

Each key is configured by the administrator with the following attributes:

| Property | Description |
|---|---|
| **Scopes** | One or more of `assets:read`, `grades:read`, `findings:read`, `data:read` |
| **Allowed targets** | Optional whitelist of target names the key may query |
| **IP allowlist** | Optional CIDR ranges; requests from other IPs are rejected |
| **Rate limit** | Per-key sliding-window RPM ceiling (default: 60 RPM) |
| **Expiry** | Optional expiration date/time after which the key is invalid |

### What happens with invalid keys

| Condition | HTTP Status | Notes |
|---|---|---|
| No `Authorization` header | `401 Unauthorized` | Include the header on every request |
| Malformed token | `401 Unauthorized` | Ensure the full `asmng_ext_...` value is sent |
| Expired key | `401 Unauthorized` | Contact your administrator to renew |
| Revoked key | `401 Unauthorized` | Contact your administrator |
| Missing required scope | `403 Forbidden` | Request scope expansion from your administrator |
| Target not authorized for key | `403 Forbidden` | Request target access from your administrator |
| Request from non-allowlisted IP | `403 Forbidden` | Ensure requests originate from your registered IPs |

## Scopes

Access to each endpoint is gated by one or more scopes. Your administrator assigns scopes when creating your key. A key may have multiple scopes.

| Scope | Grants Access To |
|---|---|
| `assets:read` | `GET /v1/ext/targets`, `GET /v1/ext/targets/{target}/assets` |
| `grades:read` | `GET /v1/ext/targets/{target}/grade`, `GET /v1/ext/targets/{target}/grade/history` |
| `findings:read` | `GET /v1/ext/targets/{target}/findings`, `GET /v1/ext/targets/{target}/findings/summary` |
| `data:read` | `GET /v1/ext/targets/{target}/results`, `GET /v1/ext/targets/{target}/results/lookup` |

Calling an endpoint without the required scope returns `403 Forbidden`.

## Rate Limiting

Two independent rate limits apply to all requests.

### IP-level limit (nginx)

The reverse proxy enforces a global per-IP limit of **30 requests per minute** with a burst allowance of 10. This limit applies regardless of which API key is used from a given IP address.

### Per-key limit (sliding window)

Each API key has an individual rate limit enforced by a 60-second sliding window. The default ceiling is **60 RPM**, but your administrator may set a lower limit for your key.

When a rate limit is exceeded, the server responds with `429 Too Many Requests`. Wait at least 60 seconds before retrying. Do not implement tight retry loops against rate-limited responses.

**Best practice:** For batch integrations, space requests evenly rather than sending them in bursts.

### List Targets

```
GET /v1/ext/targets
```

Returns a list of targets that the API key is authorized to access, along with their current grade and basic summary statistics.

**Required scope:** `assets:read`

**Query parameters:** None

**Example request:**

```bash
curl -s \
  -H "Authorization: Bearer asmng_ext_xK9mPq3r5sT7uV1wX2yZ4aBcDeFgHiJ" \
  https://asm.example.com/v1/ext/targets
```

**Example response:**

```json
[
  {
    "target": "acmecorp.com",
    "current_grade": "B",
    "current_score": 82.5,
    "last_scan": 1735689600000,
    "asset_count": 47
  },
  {
    "target": "portal.acmecorp.com",
    "current_grade": "C",
    "current_score": 71.3,
    "last_scan": 1735603200000,
    "asset_count": 12
  }
]
```

**Response fields:**

| Field | Type | Description |
|---|---|---|
| `target` | string | The target identifier (domain, hostname, etc.) |
| `current_grade` | string | Letter grade from the most recent snapshot (A–F) |
| `current_score` | number | Numeric score (0–100) from the most recent snapshot |
| `last_scan` | integer | Unix epoch milliseconds of the most recent completed scan |
| `asset_count` | integer | Number of known assets associated with this target |

**Required scope:** `data:read`

### Asset Lookup

```
GET /v1/ext/targets/{target}/results/lookup
```

Returns all scan event rows where the specified value appears as the primary discovered data. This is an **asset-centric view** — given a specific IP, domain, email address, or other artifact, it returns everything the scan engine collected about that specific value.

**Use case:** Your SOC agent flags an alert for a specific IP (`203.0.113.55`). Query this endpoint to instantly retrieve every piece of data ASM-NG has about that IP — open ports, associated hostnames, certificates, vulnerabilities, breach associations, and more — for immediate triage context.

**Required scope:** `data:read`

**Path parameters:**

| Parameter | Description |
|---|---|
| `target` | Target identifier, e.g. `acmecorp.com` |

**Query parameters:**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `value` | string | **required** | The asset value to look up, e.g. `203.0.113.55`, `alice@acmecorp.com`, `dev.acmecorp.com` |
| `limit` | integer | 200 | Maximum rows to return (max: 2000) |

**Example request:**

```bash
curl -s \
  -H "Authorization: Bearer asmng_ext_xK9mPq3r5sT7uV1wX2yZ4aBcDeFgHiJ" \
  "https://asm.example.com/v1/ext/targets/acmecorp.com/results/lookup?value=198.51.100.42"
```

**Example response:**

```json
[
  {
    "event_type": "IP_ADDRESS",
    "event_type_label": "IP Address",
    "data": "198.51.100.42",
    "source_data": null,
    "module": "sfp_dnsresolve",
```

## Error Reference

All error responses use standard HTTP status codes. The response body may contain a `detail` field with additional context.

| HTTP Status | Meaning | Common Causes | Recommended Action |
|---|---|---|---|
| `401 Unauthorized` | Authentication failed | Missing header, malformed token, expired key, revoked key | Verify the `Authorization` header is present and correct; contact your administrator if the key has expired or been revoked |
| `403 Forbidden` | Authorization denied | Missing scope for endpoint; target not authorized for key; request originating from a non-allowlisted IP | Verify your key has the required scope; confirm you are authorized for the target; check your egress IP against your registered allowlist |
| `404 Not Found` | Target has no data | The target exists in the system but has no scan data yet, or the target identifier is incorrect | Verify the target name exactly as provided by your administrator; wait for a scan to complete if the target was recently added |
| `429 Too Many Requests` | Rate limit exceeded | Per-key RPM ceiling or per-IP nginx limit reached | Wait 60 seconds before retrying; reduce request frequency; avoid burst patterns |
| `500 Internal Server Error` | Server-side error | Unexpected application error | Retry after a short delay; report the issue to your ASM-NG administrator if the error persists |
| `503 Service Unavailable` | External API disabled | Administrator has activated the kill switch for the External API | The API is temporarily unavailable; contact your administrator |

## Timestamps

All timestamp fields in API responses are **Unix epoch milliseconds** — the number of milliseconds elapsed since 1970-01-01T00:00:00Z (UTC).

To convert to seconds, divide by 1000:

```python
import datetime
ts_ms = 1735689600000
dt = datetime.datetime.utcfromtimestamp(ts_ms / 1000)
# 2025-01-01 00:00:00
```

```javascript
const ts_ms = 1735689600000;
const dt = new Date(ts_ms);
// Wed Jan 01 2025 00:00:00 UTC
```

All timestamps in the API are UTC. There is no timezone conversion applied server-side.

## Grade System

ASM-NG calculates security grades using a weighted average across eight categories. Each category has a score (0–100) and a weight; the overall score is the weighted mean. Letter grades are assigned based on the overall numeric score.

### Grade categories and weights

| Category | Weight | What It Measures |
|---|---|---|
| Network Security | 1.0 | Open ports, exposed services, firewall posture |
| Web App Security | 1.0 | Web vulnerabilities, headers, TLS configuration |
| Information Leakage | 0.8 | Exposed credentials, sensitive data in public sources |
| General Health | 0.8 | Certificate validity, DNS hygiene, overall surface cleanliness |
| External Account Exposure | 0.7 | Compromised accounts, credential exposure in breach data |
| DNS Health | 0.7 | SPF/DKIM/DMARC, zone transfer exposure, subdomain takeover risk |
| IP Reputation | 0.6 | Blacklist presence, malware hosting history, spam reputation |
| AI Security | 0.9 | LLM endpoint exposure, AI-related attack surface |

### Letter grade thresholds

| Grade | Score Range | Interpretation |
|---|---|---|
| A | 90 – 100 | Strong security posture; minimal critical exposure |
| B | 80 – 89 | Good posture with some areas for improvement |
| C | 70 – 79 | Moderate risk; notable issues present |
| D | 60 – 69 | Elevated risk; significant issues require attention |
| F | 0 – 59 | Critical risk; immediate remediation recommended |

The overall score is computed as:

```
overall_score = sum(category_score * category_weight) / sum(category_weight)
```

## Code Examples

### Python — Full working example

The following example demonstrates authentication, listing targets, and fetching the current grade for the first authorized target.

```python
import requests
import datetime

# Configuration
BASE_URL = "https://asm.example.com/v1/ext"
API_KEY = "asmng_ext_xK9mPq3r5sT7uV1wX2yZ4aBcDeFgHiJ"

HEADERS = {
    "Authorization": f"Bearer {API_KEY}",
    "Accept": "application/json",
}

session = requests.Session()
session.headers.update(HEADERS)


def health_check():
    """Verify the API is reachable (no auth required)."""
    r = requests.get(f"{BASE_URL}/health", timeout=10)
    r.raise_for_status()
    return r.json()


def list_targets():
    """Return all targets authorized for this key."""
    r = session.get(f"{BASE_URL}/targets", timeout=10)
    r.raise_for_status()
    return r.json()


def get_current_grade(target: str):
    """Return the most recent grade snapshot for a target."""
    r = session.get(f"{BASE_URL}/targets/{target}/grade", timeout=10)
    r.raise_for_status()
    return r.json()


def get_grade_history(target: str, limit: int = 12):
    """Return up to `limit` historical grade snapshots (oldest first)."""
    r = session.get(
        f"{BASE_URL}/targets/{target}/grade/history",
        params={"limit": limit},
        timeout=10,
    )
    r.raise_for_status()
    return r.json()


def get_findings_summary(target: str):
    """Return a count of validated findings grouped by type."""
    r = session.get(f"{BASE_URL}/targets/{target}/findings/summary", timeout=10)
    r.raise_for_status()
    return r.json()


def format_timestamp(ts_ms: int) -> str:
    """Convert Unix milliseconds to a readable UTC string."""
    return datetime.datetime.utcfromtimestamp(ts_ms / 1000).strftime("%Y-%m-%d %H:%M UTC")


if __name__ == "__main__":
    # Step 1: Health check
    health = health_check()
    print(f"API status: {health['status']}")

    # Step 2: List authorized targets
    targets = list_targets()
    print(f"\nAuthorized targets ({len(targets)}):")
    for t in targets:
        print(
            f"  {t['target']:<30} Grade: {t['current_grade']}  "
            f"Score: {t['current_score']:.1f}  "
            f"Assets: {t['asset_count']}  "
            f"Last scan: {format_timestamp(t['last_scan'])}"
        )

    if not targets:
        print("No authorized targets found.")
        exit(0)

    # Step 3: Fetch current grade for the first target
    first_target = targets[0]["target"]
    grade = get_current_grade(first_target)

    print(f"\nCurrent grade for {first_target}: {grade['overall_grade']} ({grade['overall_score']:.1f})")
    print(f"Snapshot taken: {format_timestamp(grade['as_of'])}")
    print("\nCategory breakdown:")
    for category, data in grade["categories"].items():
        print(
            f"  {category:<30} {data['grade']}  "
            f"({data['score']:.1f})  weight={data['weight']}"
        )

    # Step 4: Fetch findings summary
    summary = get_findings_summary(first_target)
    print(f"\nFindings summary for {first_target}:")
    print(f"  Total validated: {summary['total_validated']}")
    print(f"  FP suppression active: {summary['total_fp_hidden']}")
    print("  By type:")
    for event_type, count in sorted(summary["by_type"].items(), key=lambda x: -x[1]):
        print(f"    {event_type:<40} {count}")
```

### Python — Paginating through all assets

```python
def get_all_assets(target: str, asset_type: str = None, status: str = "CONFIRMED"):
    """Fetch all assets for a target, handling pagination automatically."""
    assets = []
    offset = 0
    limit = 100

    while True:
        params = {"limit": limit, "offset": offset, "status": status}
        if asset_type:
            params["type"] = asset_type

        r = session.get(f"{BASE_URL}/targets/{target}/assets", params=params, timeout=10)
        r.raise_for_status()
        page = r.json()

        assets.extend(page)

        if len(page) < limit:
            # Fewer results than requested means we are on the last page
            break

        offset += limit

    return assets


# Example usage
all_hostnames = get_all_assets("acmecorp.com", asset_type="INTERNET_NAME")
print(f"Total confirmed hostnames: {len(all_hostnames)}")
```

### Python — Handling errors robustly

```python
import time

def get_grade_with_retry(target: str, max_retries: int = 3):
    """Fetch grade with automatic retry on 429 and 5xx responses."""
    for attempt in range(max_retries):
        try:
            r = session.get(f"{BASE_URL}/targets/{target}/grade", timeout=10)

            if r.status_code == 429:
                print(f"Rate limited. Waiting 60 seconds before retry {attempt + 1}/{max_retries}...")
                time.sleep(60)
                continue

            if r.status_code == 503:
                print("External API is currently disabled by the administrator.")
                return None

            if r.status_code == 404:
                print(f"No data available for target: {target}")
                return None

            r.raise_for_status()
            return r.json()

        except requests.exceptions.RequestException as e:
            print(f"Request error on attempt {attempt + 1}: {e}")
            if attempt < max_retries - 1:
                time.sleep(5)

    print(f"Failed to fetch grade for {target} after {max_retries} attempts.")
    return None
```

## Getting Access

API keys for the External API are issued exclusively by the **ASM-NG administrator** for your organization. There is no self-service key provisioning.

To request access or modify an existing key, contact your ASM-NG administrator with the following information:

- **Purpose**: Brief description of your integration (e.g., "AI SOC agent — read grades and findings for triage")
- **Required scopes**: Which of `assets:read`, `grades:read`, `findings:read`, `data:read` you need
- **Targets**: Which target(s) you need access to
- **Source IPs**: The IP addresses or CIDR ranges your system will make requests from (for allowlisting)
- **Expected request volume**: Approximate requests per minute so an appropriate rate limit can be configured
- **Expiry requirement**: Whether the key needs an expiration date

Keys should be stored securely (e.g., in a secrets manager) and must not be embedded in source code or shared with unauthorized parties. If a key is compromised, notify your administrator immediately for revocation.
